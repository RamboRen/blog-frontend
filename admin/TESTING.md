# 文章管理功能测试指南

## 🎯 测试步骤

### 1. 启动后台管理系统

```bash
cd admin
npm run dev
```

访问 http://localhost:3001

### 2. 测试文章列表页

访问 `/posts` 页面，验证以下内容：

- ✅ 页面显示 3 篇示例文章
- ✅ 搜索框可以输入文字
- ✅ 状态筛选器可以切换（全部/已发布/草稿）
- ✅ 统计卡片显示正确数量
- ✅ 每篇文章有编辑和删除按钮

**测试操作：**
1. 在搜索框输入 "React"，应该只显示 React 相关文章
2. 选择"已发布"状态，应该只显示已发布的文章
3. 点击"新建文章"按钮，应该跳转到编辑页面

### 3. 测试 TipTap 编辑器

访问 `/posts/new` 页面，测试编辑器功能：

#### 工具栏测试

**文字格式化：**
1. 输入一些文字并选中
2. 点击 `B` 按钮 - 文字应该变粗
3. 点击 `I` 按钮 - 文字应该倾斜
4. 点击 `S` 按钮 - 应该添加删除线
5. 点击 `</>` 按钮 - 应该变成行内代码样式

**标题测试：**
1. 点击 `H1` 按钮 - 应该创建一级标题
2. 点击 `H2` 按钮 - 应该创建二级标题
3. 点击 `H3` 按钮 - 应该创建三级标题

**列表测试：**
1. 点击列表按钮 - 应该创建无序列表
2. 点击编号列表按钮 - 应该创建有序列表
3. 按 Enter 键应该创建新的列表项

**对齐测试：**
1. 输入一段文字并选中
2. 分别点击左对齐、居中、右对齐按钮
3. 文字应该相应改变对齐方式

**块级元素测试：**
1. 点击引用按钮 - 应该创建引用块
2. 点击代码块按钮 - 应该创建代码块
3. 点击分割线按钮 - 应该插入水平线

**插入测试：**
1. 点击图片按钮，输入 URL：`https://picsum.photos/800/400`
   - 应该插入图片
2. 点击链接按钮，输入 URL：`https://example.com`
   - 应该创建超链接

**历史操作测试：**
1. 输入一些文字
2. 按 Ctrl+Z 或点击撤销按钮 - 文字应该消失
3. 按 Ctrl+Y 或点击重做按钮 - 文字应该恢复

#### 编辑器样式测试

验证以下样式是否正确：
- ✅ 标题有明显的层级区分
- ✅ 代码块有深色背景和等宽字体
- ✅ 引用块有左侧边框和灰色文字
- ✅ 列表有正确的缩进
- ✅ 图片自适应宽度且有圆角
- ✅ 链接有蓝色和下划线

## 🐛 已知问题排查

### 问题 1：编辑器不显示

**症状：** 编辑器区域空白

**解决方案：**
1. 检查控制台是否有错误
2. 确认 TipTap 依赖已安装：`npm list @tiptap/react`
3. 重启开发服务器

### 问题 2：样式不正确

**症状：** 编辑器样式混乱或工具栏不显示

**解决方案：**
1. 确认 Tailwind CSS 配置正确
2. 检查 `index.css` 中的 ProseMirror 样式
3. 清除浏览器缓存并刷新

### 问题 3：图片无法插入

**症状：** 点击图片按钮无反应

**解决方案：**
1. 确保输入完整的 URL（包含 https://）
2. 检查图片 URL 是否有效
3. 某些网站可能禁止跨域引用

### 问题 4：移动端菜单无法打开

**症状：** 点击菜单按钮无反应

**解决方案：**
1. 检查 Layout 组件的 sidebarOpen 状态
2. 确认 z-index 层级正确
3. 检查是否有 JavaScript 错误
